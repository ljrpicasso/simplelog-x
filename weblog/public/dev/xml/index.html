XML development

XML isn't just a markup language, it's a way of life. Or something like that. Anyway, it's all I use as far as this website is concerned, and I'm a big fan of XML.

My work

  • Saxite is my 100% XSLT website framework
  • AxKit-related work is all XML-based
  • Alexandra forms is a trippy XSLT-based forms round-tripping scheme

Links

  • Monastic XML is Simon St. Laurent's brilliant analysis of how to use XML. It's a tool that can be abused and misused, but if you follow his guidance you won't go too far wrong.
  • XSLT FAQ is Dave Pawson's excellent collection of advice from xsl-list. That list itself is the go-to place for questions not answered in the FAQ.
